---

# mission

transform knowledge into exam performance.

assume:

the student already has notes.

the student already has concept understanding.

the bottleneck is:

* answer generation
* retrieval speed
* exam execution

optimize for:

1. active recall
2. retrieval practice
3. answer construction
4. exam simulation
5. marking awareness

---

# phase 1

identify:

* highest probability exam topics
* highest difficulty concepts
* most interconnected concepts
* most frequently emphasized concepts

rank:

high priority

medium priority

low priority

focus drills on high-priority concepts.

---

# phase 2

generate:

06-answer-drills/high-priority-drills.md

for every high-priority concept create:

## recall drill

example:

define X

explain Y

list properties of Z

student must answer from memory.

do not provide answer immediately.

place answer in collapsible section below.

---

## concept connection drill

example:

how does X relate to Y?

compare X and Z.

why is X required before Y?

---

## application drill

create realistic scenarios.

require concept application.

avoid pure definition questions.

---

## synthesis drill

combine multiple concepts.

example:

use A, B and C to solve the problem.

---

# phase 3

generate:

06-answer-drills/model-answers.md

for each high-priority question:

provide:

* ideal answer
* examiner expectations
* common mistakes
* partial-credit opportunities

focus on what earns marks.

not on maximum verbosity.

---

# phase 4

generate:

06-answer-drills/marking-guide.md

for each likely exam topic:

create:

## full marks answer

## average answer

## failing answer

explain why.

show the difference.

---

# phase 5

generate:

06-answer-drills/exam-simulations.md

create:

3 mock exams

easy

medium

hard

each exam should:

* resemble likely professor style
* mix concept types
* include comparison questions
* include synthesis questions
* include application questions

do not provide answers in the exam.

place answers in separate section.

---

# phase 6

generate:

06-answer-drills/rapid-fire-recall.md

create:

100 ultra-short questions.

goal:

answer each within 5-15 seconds.

examples:

define

list

identify

differentiate

state

give an example

optimize for retrieval speed.

---

# phase 7

generate:

06-answer-drills/feynman-drills.md

for each major concept:

create prompts:

explain this to:

* a child
* a high school student
* a first-year university student

identify gaps in understanding.

---

# phase 8

generate:

06-answer-drills/oral-exam-drills.md

simulate a professor asking questions.

start easy.

increase difficulty.

include follow-up questions.

---

# phase 9

generate:

06-answer-drills/final-exam-readiness-report.md

evaluate:

* strongest topics
* weakest topics
* highest risk topics
* likely exam gaps

create:

recommended revision order

recommended mock exam order

recommended final review checklist

---

# quality standards

prioritize:

* retrieval
* answer construction
* exam performance

avoid:

* passive summaries
* transcript rewriting
* unnecessary explanations

every drill should force active recall.

every answer should reflect likely grading expectations.

the output should function as a personal exam coach rather than another set of notes.
