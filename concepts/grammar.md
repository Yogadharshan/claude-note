## definition

A **grammar** G = (V, T, S, P) where:
- V = finite set of **variables** (non-terminals)
- T = finite set of **terminal symbols** (the actual alphabet)
- S ∈ V = **start symbol**
- P = finite set of **production rules** (rewriting rules)

V ∩ T = ∅ (variables and terminals are disjoint)

A grammar generates a language: L(G) = {w ∈ T* | S ⟹* w}

## intuition

A grammar is a recipe for generating strings. You start with the special symbol S, apply production rules one by one (replacing left-hand sides with right-hand sides), and continue until only terminal symbols remain. Every string reachable this way is in the language.

## why it exists

Automata recognize strings one symbol at a time. Grammars provide an alternative, generative perspective: describe what valid strings look like by specifying how they're built. Grammars are often more intuitive for complex structures (like arithmetic expressions or programming language syntax).

## grammar types (Chomsky Hierarchy)

| Type | Name | Rule Form | Language |
|------|------|-----------|---------|
| 3 | Regular | A → xB or A → x | Regular |
| 2 | Context-Free | A → α | Context-Free |
| 1 | Context-Sensitive | αAβ → αγβ | Context-Sensitive |
| 0 | Unrestricted | α → β | Recursively Enumerable |

- Type 3 is most restrictive; Type 0 is most general
- Each type is strictly more expressive than the next

## applications

- Defining programming language syntax (CFG)
- Generating test cases for parsers
- NLP: formal models of natural language structure
- Defining protocols and data formats

## advantages

- Generative: intuitively describes the structure of valid strings
- Hierarchical: naturally expresses nested and recursive structures
- Compositional: complex grammars built from simpler rules

## limitations

- Grammars can be ambiguous (multiple parse trees for one string)
- Parsing grammars can be computationally expensive depending on type

## misconceptions

- Non-terminals V are internal symbols; they never appear in generated strings
- The same language can be generated by infinitely many grammars (not unique)
- A grammar specifies structure, not meaning (semantics is separate)

## examples

G = ({S}, {a, b}, S, P) where P = {S → aSb | ab}
- Generates: L(G) = {a^n b^n | n ≥ 1}
- Derivation: S ⟹ aSb ⟹ aaSbb ⟹ aaabbb

G = ({S}, {a, b}, S, P) where P = {S → aS | bS | ε}
- Generates: L(G) = {a,b}* (all strings over {a,b})

## exam relevance
**high** — grammar definitions and derivations appear in almost every formal languages exam

## project relevance
Any compiler, interpreter, or structured data parser uses a grammar (usually CFG) as its core specification. Grammar-based code generators (yacc, bison, ANTLR) are standard tools.
