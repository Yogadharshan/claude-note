## definition

A language L is **regular** if and only if:
- ∃ a Regular Expression (RE) that describes L, OR equivalently
- ∃ a DFA that accepts L, OR equivalently
- ∃ a right or left linear grammar that generates L

All three characterizations define exactly the same class of languages.

## intuition

Regular languages are the "simplest" infinite languages. They can be recognized with finite memory (a DFA has only finitely many states). They cannot count or match balanced structures — anything requiring unbounded memory is not regular.

## why it exists

Regular languages form the foundation class in the Chomsky hierarchy. They are computationally cheapest to recognize (O(n) time, O(1) extra space beyond the input), making them ideal for lexical analysis and simple pattern matching.

## Three Equivalent Representations

```
Regular Expression ↔ DFA ↔ Linear Grammar
```
- RE → DFA: Thompson construction (RE → NFA) then subset construction (NFA → DFA)
- DFA → RE: state elimination algorithm
- DFA → Linear Grammar: each state becomes a non-terminal

## Closure Properties
Regular languages are closed under:

| Operation | Closed? |
|-----------|---------|
| Union | YES |
| Intersection | YES |
| Complement | YES |
| Concatenation | YES |
| Kleene Star | YES |
| Reverse | YES |
| Homomorphism | YES |
| Right Quotient | YES |

Regular languages are closed under ALL common Boolean and language operations.

## Non-Regular Languages (common examples)
- L = {a^n b^n | n ≥ 1} — requires counting (not regular)
- L = {w ∈ {a,b}* | w is a palindrome} — requires unbounded memory
- L = {w | w has equal a's and b's} — requires counting

Proved non-regular using the [[pumping-lemma-regular]] or [[myhill-nerode-theorem]].

## applications

- Lexical analysis: all token patterns in programming languages are regular
- Text search: grep, sed, awk — all use regular language pattern matching
- Simple protocol validation where counting is not required
- Hardware control logic (finite state machines)

## advantages

- O(n) recognition time
- Closed under all Boolean operations (complement, intersection, etc.)
- Decidable: membership, emptiness, finiteness, equivalence all decidable
- DFA minimization gives a canonical unique representation

## limitations

- Cannot express balanced/nested structures (parentheses, brackets)
- Cannot count (no a^n b^n type languages)
- Cannot express string equality or repetition patterns

## misconceptions

- NOT all finite languages are the only regular languages — infinite regular languages exist (e.g., a*)
- All finite languages are regular (trivially — construct a DFA that accepts each string via a chain of states)
- Regular languages closed under complement does NOT mean their complements are trivial to construct (might need DFA construction)

## examples

**Regular:**
- L = {w ∈ {0,1}* | w ends in 01} — RE: (0+1)*01
- L = {w | w has even length} — RE: (aa+bb+ab+ba)*... more precisely: ((0+1)(0+1))*
- L = all strings over {a,b} — RE: (a+b)*

**Not Regular:**
- {a^n b^n | n ≥ 0}
- {w | w = w^R} (palindromes)
- {a^(2^n) | n ≥ 0}
